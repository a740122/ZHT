zht.peer – ZHT Peer Implementation¶

Peers are the outside entities that each Node communicates with.

class zht.peer.Peer(node, identity, repAddr, pubAddr, sock)[source]¶

Bases: object

Construct a new Peer instance.

The synchronization process will also happen, but will take place in a spawned greenlet.

Parameters:
  • node – The local Node that owns this Peer object.
  • identity – The identity string of the remote Peer.
  • repAddr – The ZMQ address of the remote Peer’s REP socket.
  • pubAddr – The ZMQ address of the remote Peer’s PUB socket.
  • sock – A ZMQ REQ socket connected to the remote Peer’s REP socket.
_initState()[source]¶

Initialize the internal state of this Peer object.

Any Bucket synchronization that needs to happen will occur during this initialization process.

_makeRequest(req)[source]¶

Make a request to this Peer.

Parameters:req – The request to send.
Returns:The response to the request.
